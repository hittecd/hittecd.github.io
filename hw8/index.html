About this app

Introduction

For this app I chose to implement 3 different switch levels. Switch 1 iterates Left-Right, Top-Bottom, every 0.5 seconds. Switch 3 allows the user to move left and right. Finally, Switch 5 allows the user to move Up, Left, Right, and Down.

I tested each switch by performing the following calculations:

  • 7 + 3
  • 7 - 3
  • 7 * 3
  • 7 / 3
  • 3 + 7
  • 3 - 7
  • 3 * 7
  • 3 / 7

I chose these calculations because they contain every combination of navigation paths across the key pad. I tested each calculation 3 times per Switch level and test method. My test methods involved first performing the calculations with my hands, and then performing them by using my nose.

Instructions

Chose a switch level via the radio buttons underneath the calculator.

  • Switch 1: SPACE to select
  • Switch 3: SPACE to select, "a" - left, "d" - right
  • Switch 5: SPACE to select, "w" - up, "a" - left, "s" - down, "d" - right

Findings

My main take away from my tests was that the each switch level offered distinct advantages and disadvantages.

Switch 1

To begin with, the main problem I encountered with Switch 1 was that I was required to wait for the cursor to cycle around to my desired character, even if the cursor was one past the character I wanted to select. For example, in the calculation 3 * 7, the user must wait for the cursor to cycle all the way back around to * after selecting 3 even though the 3 key is 2 keys away by the shortest path. This was especially frustrating when I simply failed to press a key in time. However, when using my nose, this actually gave me time to rest my neck and glace at the screen for reference as to where I was.

Switch 3

When using Switch 3, I encountered a lesser version of the problem in Switch 1. While I could reduce the number of keys traversed by moving backwards and forwards, I still had move as many as 4 key spaces away to select a key located directly above my starting position. The ability to directly control my cursor however meant that if I overshot or undershot my desired key, I could easily correct for it. I also found that Switch 3 was the most intuitive to learn. I was able to quickly and consistently improve my calculation times when I switched to this system.

Switch 5

Finally, Switch 5 offered the most extreme pro's and con's. This stemmed from the fact that when the ability to move in any direction was fully taken advantage of, the greatest speed increases could be attained. However, to fully take advantage of Switch 5, a strong faimiliarity with the keypad was required. I often found myself scrolling all the way to the left or right when fewer keystrokes in the up or down doriection would have been more performant. This was especially true when using my nose, and most of my concentration was focused simply hitting the keys correctly. Therefore, while I was able to perform the calculations the fastest with Switch 5, those times were also less consistent than with any other system.
